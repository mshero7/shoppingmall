# 고객 :two_men_holding_hands:



## :heavy_check_mark: 회원가입

```
회원가입

1. 액터명
 고객

2. 개요 
 회원가입을 진행해 사이트에 회원으로 활동할 수 있다.

3. 사전조건
 회원가입하지 않은 고객, 즉 비회원이어야 한다.

4. 기본 흐름
  1) 회원가입 버튼을 클릭한다. (액터)
  2) 회원가입을 원하는 고객은 최초 이름과 주민번호를 입력한다. (액터)
  3) 정보들을 테이블과 비교한다. (시스템)
  4) 회원가입약관을 읽고 동의한다. (액터)
  5) 원하는 회원 아이디를 입력한다. (액터)
  6) 비밀번호와 비밀번호확인을 입력한다. (액터)
  7) 이름, 생년월일, 성별, 주소지, 이메일 등 필수입력 항목을 작성한다. (액터)
  8) 직업, 기혼여부, 뉴스레터 수신여부 등 선택입력 항목들을 작성한다. (액터)
  9) 회원가입 데이터를 회원(member)테이블에 저장한다. (시스템)
  
5. 대체 흐름
  2-a) 이미 가입한 회원일 경우 회원가입을 진행하지 못하도록 한다.
  4-a) 회원가입약관들에 대해 동의하였는지 확인한다.
  5-a) 중복되는 아이디는 없는지 확인한다
  6-a) 비밀번호와 비밀번호확인의 입력이 같은지 확인한다.
  7-a) 이미 아이디가 존재한다면 다른 아이디를 입력받게끔 한다.
  8-a) 비밀번호와 비밀번호확인이 서로 불일치하면 같은 비밀번호를 입력하게 한다.
  12-a) 필수입력 항목들이 모두 작성되어있는지 확인한다.
```



## ✔️  로그인

```
로그인

1. 액터명
 고객

2. 개요 
 회원가입을 했던 고객이라면 아이디와 비밀번호를 통해 로그인을 할 수 있다.

3. 사전조건
 회원가입을 한 고객이어야 한다.

4. 기본 흐름
  1) 페이지 상단의 로그인 버튼을 클릭한다.(액터)
  2) 아이디와 비밀번호를 입력할 수 있는 페이지를 출력한다.(시스템)
  3) 아이디를 입력한다.(액터)
  4) 비밀번호를 입력한다.(액터)
  5) 아이디와 비밀번호를 조회한다.(시스템)
  6) 쇼핑몰 메인 페이지로 이동시킨다.(시스템)
  
5. 대체 흐름
  4-a) 아이디가 일치하지 않는 경우 '회원정보가 일치하지 않습니다. 다시 로그인 해주세요' 라는 알림창	을 보여준다.
  4-b) 아이디는 일치하나 비밀번호가 일치하지 않는 경우 '회원정보가 일치하지 않습니다. 다시 로그인 		해주세요'라는 알림창을 보여준다.
```

## ✔️  주문결재

```

1. 액터명
 고객

2. 개요

3. 사전조건
 회원이라면 로그인을, 비회원이라면 이름과 전화번호인증을 통해 주문을 시도해야한다.

  3) 주문입력 페이지에 개인정보를 입력한다. (액터)
  4) 주문입력 페이지에 배송지정보를 입력한다. (액터)
  5) 주문입력 페이지에 결재수단를 입력한다. (액터)
  6) 주문입력 페이지에 주문약관동의를 입력한다. (액터)
  7) 결재하기 버튼을 누른다. (액터)
  8) 주문정보를 저장한다. (시스템)
  
5. 대체 흐름
  3-a) 회원인 경우 회원정보를 바탕으로 작성된다.
  4-a) 회원인 경우 회원정보를 바탕으로 작성된다.
  5-a) 사용자가 입력한 결재수단에 주문금액에 맞는 돈을 인출할 수 있는지 확인한다.  
  7-a) 주문약관동의에 동의되지 않은 경우 동의해야 한다는 알림창을 띄운다.
  7-b) 결재수단이 선택되지 않은 경우 결재수단을 선택하게 한다.
  7-c) 배송지가 입력되지 않은 경우 배송지를 입력하게 한다.

# 관리자 :guardsman:



## ✔️  회원 관리

```
회원관리

1. 액터명
 관리자

2. 개요 
 관리자는 오래된고객의 정보를 지우거나 마일리지 정보 등을 갱신해줄 수 있다.

3. 사전조건
 관리자로 로그인이 되어있어야 하며, 최고관리자만이 고객의 정보에 접근할 수 있다.

4. 기본 흐름
  1) 관리자 페이지로 이동한다.(액터)
  2) 관리자 페이지를 보여준다.(시스템)
  3) 회원관리 페이지로 이동한다.(액터)
  4) 회원관리 페이지를 보여준다.(시스템)
  5) 회원을 선택해 수정을 시도한다.(액터)
  6) 수정에 대해 다시 한번 확인시키는 알림창을 띄워준다.(시스템)
  7) 수정 확인 버튼을 누른다.(액터)
  8) 수정된 내용에 대해 반영한다.(시스템)
  
5. 대체 흐름
 5-a) 회원삭제는 회원의 정보가 삭제되는 것을 의미한다.
```



## ✔️  배송 관리

```
배송 관리

1. 액터명
 관리자

2. 개요 
 관리자는 고객들의 주문내역들에 대해 관리할 수 있다. 주문내역에 대한 관리의 범위는 기간, 상품, 입금상태, 입금은행, 결재카드, 고객구분으로 관리할 수 있다.

3. 사전조건
 관리자 로그인을 한 상태여야 하며, 고객의 주문정보내역이 존재해야 한다.

4. 기본 흐름  
  1) 관리자 페이지로 이동한다.(액터)
  2) 관리자 페이지를 보여준다.(시스템)
  3) 배송관리 페이지로 이동한다.(액터)
  4) 배송관리 페이지를 보여준다.(시스템)
  5) 원하는 검색범위에 대해 입력한다. (액터)
  6) 원하는 검색조건들에 만족하는 주문내역정보를 페이지에 보여준다. (시스템)
  7) 주문내역에 갱신할 데이터들에 대해 입력해준다.(액터)
  8) 액터가 갱신한 내역을 반영한다. (시스템)
  
5. 대체 흐름
```



## ✔️  카테고리 관리

```
카테고리 관리

1. 액터명
 관리자

 관리자는 상품분류에 활용될 카테고리를 등록, 및 수정할 수 있다.

3. 사전조건
 관리자 로그인을 선행해야 한다.

  4-a) 카테고리 분류는 대분류(1차) - 중분류(2차) - 소분류(3차) - 상세분류(4차)로 나뉜다.
  5-a) 상위 카테고리가 없으면 단일 카테고리(대분류)를 추가할 수 있다.
